# B.1 Рекомендации по следующим шагам (консультация MCP, 2025-03-08)

Цель: медиана total_reward ≥71 за 50 эп. (seed 42). Текущий лучший: **12.0** (steps=5, night=200, dopamine-shaping).

## Приоритетный список (по консультации generalPurpose)

1. **Population coding входа (макс. приоритет)**  
   Сейчас в мозг идут 4 скаляра; в genesis-agi каждое наблюдение кодируется полем рецепции (64 бита, ~5 активных нейронов на переменную). **Сделать:** в Python развернуть 4 переменные в вектор активностей (16–64 нейронов на переменную, гауссово поле как `encode_population` в genesis-agi) и передавать вектор длины 64–256; Rust уже поддерживает `input.len()` и дополняет нулями до n.

2. **Inertia по весам** — **реализовано**  
   В `night_phase_plasticity` добавлена кривая по рангу |weight|: 16 уровней, сильные связи меняются слабее. См. sifs_genesis_hybrid.rs, константа `inertia_curve`, ранг `weight >> 13`.

3. **Частота Night phase и конец эпизода**  
   При night=200 и steps=5 пластичность раз в 40 шагов окружения; в коротких эпизодах обновлений мало. **Варианты:** уменьшить night_interval (80–120) и/или вызывать night phase по флагу «конец эпизода» с текущим reward.

4. **Усиление reward_scale и dopamine**  
   Сейчас `reward_scale = 1.0 + reward * 0.5`; можно увеличить коэффициент или усилить штраф за pole_av в dopamine_shaped.

5. **Больше эпизодов, метрика по последним 50**  
   Гонять 200–500 эп., медиану считать по **последним** 50 — так пластичность успеет накопиться при сохранении метрики «медиана по 50 эп.»

6. **Раздельный readout left/right (моторные пулы)**  
   Два score по непересекающимся наборам нейронов, action = argmax(left, right) или по разности — по образцу genesis-agi (64+64).

7. **Число нейронов после population coding**  
   После п.1 оставить 1k или поднять до 2k, чтобы проекция входа покрывала достаточную долю сети.

## Порядок внедрения

Сначала **population coding (1)** и **inertia (2 — сделано)**; затем 3–4; 5 — для честной оценки; 6–7 по результатам.

## Контракт входа (Rust)

В `BrainRunner::step` вход дополняется нулями до n: `.chain(std::iter::repeat(0).take(n.saturating_sub(input.len())))`. Передача вектора длины 64–256 из Python уже поддерживается; нужно в run_cartpole_agent.py заменить normalize_obs на population-encoded вектор и передавать его в payload.input.
