# Чекпоинтинг и версионирование весов (§7 плана)

Базовая схема для сохранения и отката состояния мозга. Конкретная реализация в коде — в рамках фазы B/C; до завершения фазы D схема должна быть реализована и соответствовать этому описанию.

---

## Цель

- Воспроизводимость: фиксировать состояние весов (и при необходимости сом) в ключевые моменты.
- Откат при деградации: подставить веса из предыдущего чекпоинта, если метрика падает ниже порога.
- A/B сравнение: запускать два набора весов (например с I_SIFS и без) без перезапекания.

---

## Минимум (что сохранять и когда)

| Элемент | Описание |
|--------|-----------|
| **Когда** | После Night Phase или каждые N тиков (параметр конфига). Опционально: по явному сигналу (API/файл-триггер). |
| **Что** | Состояние весов по шардам; при необходимости — сомы (если участвуют в пластичности). Формат: бинарный дамп, совместимый с загрузкой в тот же тип шарда (CPU/CUDA). |
| **Именование** | Метка времени или версия: например `weights_<task>_<timestamp>.bin` или `weights_v<N>.bin`. Связь с задачей/средой — см. [CATASTROPHIC_FORGETTING.md](CATASTROPHIC_FORGETTING.md) (отдельные чекпоинты на задачу). |
| **Где хранить** | Локальный каталог (например `checkpoints/` относительно рабочей директории); при деплое — конфигурируемый путь. |

---

## Откат

- **Критерий деградации:** метрика среды (например медиана CartPole score за последние K эпизодов) падает ниже порога X (X задаётся в конфиге или плане среды).
- **Действие:** остановить обучение/инференс на текущих весах; загрузить веса из последнего «хорошего» чекпоинта (по времени или по метке версии). Кто триггерит: оркестратор (runner среды) или отдельный монитор.
- **Логирование:** при откате записать в лог причину (метрика &lt; X), путь к загруженному чекпоинту.

---

## A/B весов

- Два набора весов (например чекпоинт A с I_SIFS, чекпоинт B без) загружаются в два экземпляра мозга (или переключение по конфигу).
- Сравнение метрик без перезапекания: один и тот же поток наблюдений подаётся в оба; результаты сравниваются по выбранной метрике (score, spike_count, и т.д.).
- Реализация: либо два процесса/узла, либо один процесс с переключением набора весов между сериями эпизодов (реализация в рамках фазы B/C).
